允许触发构建的目标分支（或标签）的正则表达式匹配，如果为空，则所有的分支（或标签）都可以触发构建。

* 只提供了针对分支的正则表达式时会导致标签推送被过滤，为了确保标签推送被执行，需要在正则中包含以 ^refs\/tags\/ 开头的正则表达式, 具体请见下面的示例.

比如:

# 允许包含 debug 或 release 的分支名触发构建: (.*debug.*|.*release.*) 
# 过滤所有包含  `Release` 或者 `release` 的分支名触发构建: ^(?:(?![R|r]elease).)*$ 
# 允许所有标签和 master 分支触发构建：(^refs\/tags\/.+)|master
